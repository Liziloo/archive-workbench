**Purpose:** Defines how our internal data maps to the Omeka S API.
*   **Dublin Core Mapping:**
    *   `Canonical_Title` -> `dcterms:title`
    *   `Verified_Date` -> `dcterms:date`
    *   `Synthesized_Description` -> `dcterms:description`
*   **Resource Templates:** Which Omeka S template to use (e.g., "Family Photograph" vs. "Legal Document").
